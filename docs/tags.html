Using the Tag members

Nearly every class in liblacewing has associated functions *_tag and *_set_tag (or two versions of tag in the C++ interface).

The tag is initialized to 0 when an instance of a class is created, but is then left alone by the library. You are free to use it to store your own pointer (perhaps extending a lacewing::server_client with your own client-specific state, for instance).

Most of the handlers that introduce new instances of classes (such as server::on_connect) will state that they do so in the documentation, and when the last time the instance will be seen is (in this case, on_disconnect). This is a good time to free any data stored in Tag.

struct ClientState
{
    /* Your data here */
};

void onConnect (lacewing::server, lacewing::server_cliet Client)
{
    Client->tag(new ClientState);
}

void onDisconnect (lacewing::server, lacewing::server_client Client)
{
    delete ((ClientState *) Client->tag());
}

void onReceive (lacewing::server, lacewing::server_client Client)
{
    ClientState &State = *(ClientState *) Client->tag(); /* Guaranteed to be valid */

}
